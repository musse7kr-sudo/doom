Top Ten Mistakes of Web Design

Since my first attempt in 1996, I have compiled many top-10 lists of the biggest mistakes in Web design.
See links to all these lists at the bottom of this article.
This article presents the highlights: the very worst mistakes of Web design. (Updated 2011.)

1. Bad Search
Overly literal search engines reduce usability in that they're unable to handle typos, plurals, hyphens, and other variants of the query terms.
Such search engines are particularly difficult for elderly users, but they hurt everybody.
A related problem is when search engines prioritize results purely on the basis of how many query terms they contain, rather than on each document's importance.
Even though advanced search can sometimes help, simple search usually works best, and search should be presented as a simple box.

2. PDF Files for Online Reading
Users hate coming across a PDF file while browsing, because it breaks their flow.
Even simple things like printing or saving documents are difficult because standard browser commands don't work.
Layouts are often optimized for a sheet of paper, which rarely matches the size of the user's browser window.
PDF is great for printing and distributing manuals and other big documents. Reserve it for this purpose and convert any information that needs to be browsed or read on the screen into real web pages.
Detailed discussion of why PDF is bad for online reading

3. Not Changing the Color of Visited Links
A good grasp of past navigation helps you understand your current location.
Links are key to this process.
Users can tell which pages they've already visited if visited links change color.

  • Usability implications of changing link colors
  • Guidelines for showing links

4. Non-Scannable Text
A wall of text is deadly for an interactive experience. Intimidating. Boring. Painful to read.
Write for online, not print.
To support scannability, use:

  • Subheads
  • Bulleted lists
  • Highlighted keywords
  • Short paragraphs
  • The inverted pyramid
  • A simple writing style
  • De-fluffed language

5. Fixed Font Size
CSS can disable a Web browser's "change font size" button.
About 95% of the time, this fixed size is tiny, reducing readability for most people over 40.
Respect user preferences and let them resize text.

6. Page Titles With Low Search Engine Visibility
Search is the main way users discover websites.
The humble page title is crucial to attract visitors and help users locate pages.
Use microcontent effectively.
Start titles with important information-carrying words, not "The" or "Welcome to".

7. Anything That Looks Like an Advertisement
Users ignore elements that look like ads.
Avoid designs that resemble advertisements.
Examples:

  • Banner blindness
  • Animation avoidance
  • Pop-up purges

8. Violating Design Conventions
Consistency is key.
Users rely on prior experience to navigate your site.
Jakob's Law: "Users spend most of their time on other websites."
If your site behaves unexpectedly, users feel insecure.

9. Opening New Browser Windows
Opening new windows disrupts users.
They may not notice and get confused with the Back button.
Links should replace the current page instead of opening new windows unnecessarily.

10. Not Answering Users' Questions
Web users are goal-driven.
Failing to provide clear information frustrates users.
The worst example is not listing the price of products.
This applies to both B2C and B2B sites.
